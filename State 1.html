Lesson 1: Introduction to the Web and HTML

The Basics of the Web and HTML

- The World Wide Web is a collection of HTML documents
- HTML is Hypertext Mark-up Language
- HTML is the base for almost all pages on the web even if they have photos, video's, etc
- Hyperlinks connect HTML pages together to form the web
- The web was invented in the early 1990s and has about 30 billion pages today

The Main Conponents that Make Up the Web

- You and me - Input commands/requests to the computer
- Computer with a web browser - Send requests to servers and display results
- Internet - Group of networks that links computers and servers
- Servers - Sends HTML documents to computers upon request

HTML Elements and Tags

- HTML Element Format - <Opening tag> Content <closing tag>
- The opening tag, content and closing tag are an element
- The closing tag name is the same as the opening tag but with a / preceeding the name
- Most elements have a closing tag but a few do not and are called empty elements
- Tags cover such items as line return, text size/formating, and many others
- Tags can have attributes like adding a url link to the text
- Two types of tags, inline and block
- Inline tags end the line; block tags make an invisible box around the text
- Example tags are <em> which is emphasis, <br> is line return, <img> is image,and <a> is anchor

HTML Miscellaneous

- Current HTML version is Version 5
- The Head of an HTML page contains random stuff and metadata
- The Body of an HTML page contains the viewable content
- Computers only do what they are togetherld to do

Useful Web and HTML Links

- World Wide Web Consortium

- Scratchpad
- HTML Color Picker

Lesson 3: Adding CSS Style to HTML Structure

HTML Structure Required for CSS

- CSS is a separate file from the HTML file and must be referenced by the HTML file
- The <link> tag is used to do this
- The HTML file should start out with <!DOCTYPE html> tag
- Always indent all lines withn html for readability
- Add the <link> tag of the CSS file within the HTML <head> section
- After the head tag place all the site content in the <body> section and continue to indent

CSS Deep Dive

- CSS is for creating style on a page using repitition instead of having to code style into each element in the HTML file
- CSS rules are Cascading
- This means a rule (like font type) will apply to all elements in the least specific part of the tree if nothing else is done
- For example if a rule is set for the body then all boxes within a body would inherit that font
- But you can specifically set rules lower in the tree with CSS if you want, for example one box in the tree may need a differnet font type and you can set it only for that box

CSS Selectors

- A CSS sheet uses a Selector to define what elements of an HTML file the particular CSS section will apply too
- Selectors can be linked to specific HTML tag types (e.g. all paragraphs)
- A more efficent way to use selectors is by class name
- The HTML concent is defined with a user defined class name(should be descritive), the selector uses the class name to apply the required style to everything that has that class name
- The format in CSS to call a class in an HTML doc is .classname { format settings}
- Multiple selectors can be combined
- One side note, CSS only changes how an HTML element looks on the page and not the element itself, its still vital to pick the right HTML for the job required

The Box Revisited

- Correct box design is one of he most critial parts of creating a webpage
- The box is made up of four components, Margin, Border, Padding, and Content
- The Content contains the text or image for the box and is at the center; protected by outer layers
- The padding clears an area around the content and inherites the background color of the box
- The border is around the padding and also has the color of the box
- The margin is the outer most layer and provides a buffer between other boxes and has no color
- Each part of the box can be changed in size using CSS
- Sum of the Box Size = content+padding+border+margin
- Size can be defined in pixels or percentage, the latter allows for better viewing of box as the size of the screen changes (will adjust to size of screen)
- CSS can alter the position of the box
- Flexbox is a key tool to allow multiple boxes on the same line of the page
- Some key design best practices with boxes:

- Use the Developer Tools section of the browser to tweak box design on the fly (then add final changes to the CSS file)
- Look for natual boxes
- Look for repeated style and semantic elements to combine with classes
- Write HTML first and then apply style (actually build page on paper or graphics program before this)
- Reiterate over the page and fix it over multiple interations
- Test HTML and CSS to ensure they work on all browsers
- This is done by following standards
